# V2 Compose Engine — 3-Layer Storyboard Architecture

**Date:** 2026-04-05
**Status:** Approved (brainstorming)
**Depends on:** MVP Production Pipeline (feat/mvp-production-pipeline)

## 1. Problem Statement

The MVP compose stage plays one continuous source video segment with narration on top. This produces watchable but low-quality content — no visual variety, no overlays, no interleaving of clips with graphics. The script already contains rich markers (`[CLIP]`, `[OVERLAY]`, `[PAUSE]`) that are ignored during composition.

Additionally, the pipeline only supports YouTube video sources. We need to support web articles and manual input. We also need Shorts/Reels output (9:16, 30-60s) from the same source material.

### Goals

- Separate knowledge (facts) from narrative (directing) from visuals (rendering)
- Make each layer independently reviewable and editable
- Enable A/B testing of narrative approaches on the same facts
- Support multiple source types (YouTube, web articles, manual)
- Support multiple output formats (full video 16:9, Shorts 9:16)
- Generate multiple Shorts from one source by cherry-picking facts
- All human interaction through Claude Code agent skills or scripts — no raw file editing

## 2. Architecture: Three Layers

```
┌─────────────────────────────────────────────────────┐
│  SOURCE ADAPTERS                                     │
│  YouTube transcript │ Web article │ Manual input      │
└──────────────┬──────────────────────────────────────┘
               ▼
┌─────────────────────────────────────────────────────┐
│  LAYER 1: KNOWLEDGE (knowledge.json)                 │
│  Facts, entities, timeline, context bridges          │
│  Persisted. Human-editable. Source of truth.         │
│  Enrichable (Wikipedia, news, manual additions).     │
└──────────────┬──────────────────────────────────────┘
               ▼  (AI generates, human reviews/edits)
┌─────────────────────────────────────────────────────┐
│  LAYER 2: STORYBOARD (storyboard.json)               │
│  Scene-by-scene: narration + visual type + overlays  │
│  Regenerable. A/B testable. Cheap to produce.        │
│  Same format for full video and Shorts.              │
└──────────────┬──────────────────────────────────────┘
               ▼  (TTS + visual renderers)
┌─────────────────────────────────────────────────────┐
│  LAYER 3: COMPOSE ENGINE                             │
│  Per-scene rendering → concatenation → subtitles     │
│  Modular visual renderers, swappable per scene.      │
│  Supports 16:9 and 9:16 output.                     │
└─────────────────────────────────────────────────────┘
```

## 3. Layer 1: Knowledge Base

File: `output/projects/{id}/knowledge.json`

### Schema

```json
{
  "meta": {
    "source_type": "youtube | web | manual",
    "source_url": "https://...",
    "title": "Original title",
    "locale": "zh-TW",
    "created_at": "ISO timestamp",
    "updated_at": "ISO timestamp"
  },
  "facts": [
    {
      "id": "f1",
      "text": "Factual statement extracted from source",
      "timestamp": "M:SS or null for non-video sources",
      "source": "transcript | web | manual | enrichment",
      "verified": false,
      "tags": ["biology", "octopus"]
    }
  ],
  "entities": [
    {
      "id": "e1",
      "name": "Entity name",
      "role": "Role or description",
      "details": "Additional context"
    }
  ],
  "timeline": [
    {
      "time": "M:SS or sequential order",
      "event": "What happened",
      "facts": ["f1", "f2"]
    }
  ],
  "context_bridges": [
    "Cultural context the target audience needs"
  ]
}
```

### Behavior

- Generated by the Analyze stage from any source adapter output
- Facts have IDs referenced by storyboard scenes (`facts_ref`)
- `verified` field defaults to false — human sets to true after review
- `source` tracks provenance: transcript, web scrape, manual input, enrichment pipeline
- `tags` on facts enable Shorts selection (filter by topic, score by interest)
- Editing a fact does NOT auto-regenerate storyboard — human triggers regeneration explicitly

### Future: Enrichment Pipeline

A separate stage (not in this spec) that takes Layer 1 keywords and crawls Wikipedia, news articles, etc. to add more facts with `source: "enrichment"`. The architecture supports this by design — enrichment just appends to the `facts` array.

## 4. Layer 2: Storyboard

File: `output/projects/{id}/storyboard.json` (or `storyboard_v{N}.json` for A/B versions)

### Schema

```json
{
  "version": 1,
  "format": "standard | short",
  "target_duration_sec": 720,
  "aspect_ratio": "16:9 | 9:16",
  "scenes": [
    {
      "id": "s1",
      "section": "hook | context | rising | climax | aftermath | analysis | content | punchline",
      "narration": "Narration text for this scene in target locale",
      "narration_est_sec": 8,
      "facts_ref": ["f1", "f2"],
      "visual": {
        "type": "clip | text_card | map | namecard | generated_image | slide | still_frame",
        "...type-specific fields..."
      },
      "overlay": {
        "type": "title | text | namecard",
        "text": "Overlay content"
      },
      "pause_after_sec": 0
    }
  ]
}
```

### Visual Type Definitions

**clip** — Extract segment from source video
```json
{"type": "clip", "source": "primary", "start_sec": 90, "end_sec": 105}
```

**text_card** — Solid background with styled text
```json
{"type": "text_card", "text": "重要資訊", "background": "#1a1a2e", "font_size": 48}
```

**map** — Location map (static image)
```json
{"type": "map", "query": "Chicago, Illinois", "style": "satellite"}
```

**namecard** — Lower-third style name introduction
```json
{"type": "namecard", "name": "Mike Lorber", "role": "Sky 5 Helicopter Reporter"}
```

**generated_image** — AI-generated image via DALL-E
```json
{"type": "generated_image", "prompt": "Aerial view of highway chase at night", "style": "cinematic"}
```
Cost: ~$0.04 per image via OpenAI API.

**slide** — PPT-style presentation slide
```json
{"type": "slide", "title": "Auto Mode Architecture", "bullets": ["Permission system", "Tool selection"], "image_prompt": "minimal diagram of AI agent loop"}
```

**still_frame** — Single frame from source video, shown as static image
```json
{"type": "still_frame", "source": "primary", "timestamp_sec": 120}
```

### Overlay Types

Overlays render on top of the visual. Optional per scene.

- **title** — Centered large text (episode title)
- **text** — Lower-third bar with text (facts, location info)
- **namecard** — Name + role lower-third

### Format: Standard vs Short

**Standard** (full video):
- `format: "standard"`, `aspect_ratio: "16:9"`
- Sections: hook → context → rising → climax → aftermath → analysis
- Target: 10-15 minutes
- 15-25 scenes

**Short** (Shorts/Reels):
- `format: "short"`, `aspect_ratio: "9:16"`
- Sections: hook → content → punchline
- Target: 30-60 seconds
- 2-4 scenes
- Visual types lean toward generated_image, text_card, slide (more variety in less time)
- Clips center-cropped from 16:9 to 9:16

### A/B Testing

Multiple storyboard versions from the same knowledge base:
- `storyboard.json` (default)
- `storyboard_v2.json` (different tone)
- `storyboard_v3.json` (different scene order)

Generated by re-running the Direct stage with different parameters (tone, emphasis, target duration).

### Shorts Selection

To generate Shorts from a full knowledge base, the AI scores each fact on:
- **Standalone interest** — understandable without context?
- **Surprise factor** — counterintuitive > obvious
- **Visual potential** — can we show something compelling?
- **Brevity** — explainable in 15 seconds?

Top-scoring facts each get their own Short storyboard. Multiple Shorts from one source.

## 5. Layer 3: Compose Engine

### Rendering Pipeline

```
For each scene in storyboard.scenes:
  1. Render visual → scene_{id}_visual.mp4
     (dispatch to appropriate renderer based on visual.type)
  2. Composite overlay on top → scene_{id}_overlaid.mp4
     (if scene.overlay is not null)
  3. Get narration audio → scene_{id}.mp3
     (from TTS stage, matched by scene ID)
  4. Combine visual + audio, trim to narration duration → scene_{id}_final.mp4
  5. Append pause gap if pause_after_sec > 0

Concatenate all scene finals → raw.mp4
Burn subtitles (full SRT from TTS timings) → final_{locale}.mp4
```

### Visual Renderers

Each renderer implements:
```python
class VisualRenderer(ABC):
    async def render(self, scene: dict, duration_sec: float,
                     aspect_ratio: str, work_dir: Path) -> Path:
        """Render visual for a scene. Returns path to .mp4 segment."""
```

| Renderer | Implementation |
|----------|---------------|
| `ClipRenderer` | FFmpeg extract. Validate start/end against source duration. Center-crop for 9:16. |
| `TextCardRenderer` | FFmpeg drawtext on solid color background. Noto Sans CJK TC font. |
| `MapRenderer` | Static Maps API or placeholder with location text. Convert PNG → video. |
| `NamecardRenderer` | FFmpeg drawtext with semi-transparent bar. |
| `ImageRenderer` | OpenAI DALL-E API → PNG → FFmpeg to video segment. Cache generated images. |
| `SlideRenderer` | Render title + bullets + optional image as layout. HTML→screenshot or FFmpeg drawtext. |
| `StillFrameRenderer` | FFmpeg extract single frame, loop to duration. |

### Overlay Compositing

Overlays render as transparent PNGs via FFmpeg drawtext, then composited on top of the visual using FFmpeg overlay filter. Position:
- `title` → centered vertically and horizontally
- `text` → lower-third bar (bottom 20% of frame)
- `namecard` → lower-third bar

### Subtitle Style

Both full video and Shorts use bottom-bar subtitle style (Noto Sans CJK TC, font size 24). Same as current implementation.

**Future upgrade:** Center-screen word-by-word highlight for Shorts (CapCut style). See `docs/future-tasks.md`.

### Aspect Ratio Handling

- 16:9 (1280x720) — standard YouTube video
- 9:16 (1080x1920) — Shorts/Reels

For 9:16, clip renderer center-crops from source 16:9. Other renderers generate natively at 9:16 resolution.

## 6. Source Adapters

### YouTube Adapter (exists, refactor)

Current acquire stage. Refactor to output normalized format that the analyze stage consumes.

```
Input: YouTube URL
Output: video.mp4, transcript text, metadata (title, channel, views, duration)
```

### Web Article Adapter (new)

```
Input: URL
Process: Fetch page → extract article text (readability/trafilatura) → extract metadata
Output: article text, metadata (title, author, publish date, source)
```

No video download. Layer 1 gets populated from article text instead of transcript. Compose stage uses generated_image, text_card, and slide visuals (no source clips).

### Manual Adapter (future)

User provides facts directly. Minimal implementation — just creates a knowledge.json from user input.

## 7. Pipeline Stages (Updated)

```
ACQUIRE (source adapter) → ANALYZE → DIRECT → [HUMAN GATE] → TTS → COMPOSE
```

| Stage | Input | Output | Changes from MVP |
|-------|-------|--------|-----------------|
| Acquire | URL + source_type | video/article + transcript/text | Add web adapter, normalize output |
| Analyze | Transcript/text | `knowledge.json` (Layer 1) | New output format, replaces story_structure + knowledge_graph dicts |
| Direct | `knowledge.json` + format + tone | `storyboard.json` (Layer 2) + `script.md` | **New stage**, replaces scriptwrite. Generates storyboard, derives script from it. |
| TTS | `script.md` | Audio segments + SRT | No change |
| Compose | `storyboard.json` + audio + source video | `final.mp4` | **Major rewrite**: scene-by-scene rendering with pluggable visual renderers |

### Direct Stage (replaces Scriptwrite)

The Direct stage is the creative engine. It takes Layer 1 and produces Layer 2.

**Inputs:**
- `knowledge.json` (Layer 1)
- Format: standard or short
- Tone: dramatic, educational, humorous, suspense (configurable)
- Target locale
- Target duration

**For standard format:** Claude generates a full storyboard with hook → context → rising → climax → aftermath → analysis structure.

**For short format:** Claude selects the best standalone fact from Layer 1 and generates a 3-scene storyboard: hook → content → punchline.

**For shorts batch:** Claude scores all facts, returns top N, generates N separate storyboard files.

**The script.md is derived** from the storyboard by concatenating `scene.narration` fields. This is what TTS reads. It is NOT a separate creative artifact — it's a view of the storyboard.

## 8. Agent Skills

Skills are Claude Code custom commands for conversational pipeline interaction.

### /produce

Full pipeline from URL to video.

```
/produce <url> [--locale zh-TW] [--format standard|short] [--source-type youtube|web]
```

Runs: acquire → analyze → shows Layer 1 summary → asks for approval → direct → shows storyboard → asks for approval → TTS → compose.

### /shorts

Generate multiple Shorts from a source.

```
/shorts <url-or-project-id> [--count 3] [--locale zh-TW]
```

If URL: runs acquire + analyze first. Then scores facts, presents top N candidates, asks which to produce. Generates and renders each Short.

### /evaluate-video

Score a video for porting potential. (Already exists.)

```
/evaluate-video <youtube-url>
```

### /knowledge

Review and edit Layer 1.

```
/knowledge [show|edit|verify] [--project-id N]
```

- `show` — summarize facts, entities, timeline
- `edit` — modify a fact, add context, add new facts
- `verify` — mark facts as verified

### /storyboard

Review and modify Layer 2.

```
/storyboard [show|swap-visual|regenerate|compare] [--project-id N]
```

- `show` — display scene-by-scene breakdown
- `swap-visual` — change visual type for a scene
- `regenerate` — create new storyboard version from same Layer 1 (different tone/angle)
- `compare` — side-by-side two storyboard versions

### /render

Compose video from storyboard.

```
/render [--project-id N] [--version V] [--short N]
```

### /status

Show project state.

```
/status [--project-id N]
```

Shows: which layers exist, review status, rendered outputs, storyboard versions.


## 10. Cost Estimate

### Per full video (standard, 12 min)
| Component | Cost |
|-----------|------|
| Analyze (Claude Sonnet → knowledge.json) | ~$0.10 |
| Direct (Claude Sonnet → storyboard) | ~$0.10 |
| TTS (edge-tts) | $0 |
| Generated images (2-3 per video) | ~$0.12 |
| Compose (FFmpeg, local) | $0 |
| **Total** | **~$0.32/video** |

### Per Short (45 sec)
| Component | Cost |
|-----------|------|
| Direct (Claude Sonnet, short storyboard) | ~$0.03 |
| TTS (edge-tts) | $0 |
| Generated image (1 per short) | ~$0.04 |
| Compose (FFmpeg, local) | $0 |
| **Total** | **~$0.07/short** |

If knowledge.json already exists (from full video), Shorts are very cheap — just the Direct + render cost.

## 11. Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Storyboard format | JSON | CRUD via agent/scripts only, diffing, machine-parseable |
| Knowledge layer persistence | File-based per project | Simple, no DB needed, git-trackable |
| Script.md derivation | Generated from storyboard, not separate artifact | Single source of truth for narrative |
| Subtitle style | Bottom-bar for both formats (v1) | Simpler, upgrade Shorts to center-screen later |
| Shorts from full video | Score facts, generate separate storyboards | Each Short is independent, same rendering pipeline |
| Source adapter pattern | Same pipeline, different input adapter | Reuse all layers for any content type |
| Visual renderer pattern | One renderer class per visual type | Swappable per scene, testable independently |
| Overlay compositing | Transparent PNG + FFmpeg overlay filter | Standard approach, works with all visual types |
| A/B testing | Multiple storyboard_vN.json files | Simple versioning, no DB needed |
| Human interaction | Agent skills wrapping CLI | Conversational, no raw file editing |
