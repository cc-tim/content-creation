# Channel Outro Design

**Date:** 2026-04-27  
**Status:** Approved for implementation

## Overview

A per-channel outro clip that appends automatically to every video before upload. The outro is pre-rendered once per channel profile and stored alongside the channel config. The publish preflight attaches it with FFmpeg concat — no per-video manual step required.

## Design Decisions

| Decision | Choice | Reason |
|---|---|---|
| Visual style | Warm & Cozy (amber gradient) | Matches 理想父母's empathetic, parenting tone |
| Duration | 6s animated + 14s static hold = 20s total | YouTube requires 20s for end screen cards |
| Content layout | Profile First (avatar is hero) | Personal, face-forward, builds channel identity |
| Music | 4-note C major arpeggio (FFmpeg sine) | Distinctive brand ident, zero licensing, reproducible |
| Integration point | Publish preflight | Aligns with "before upload, during metadata check" workflow |

## Animation Sequence

| Time | What happens | Music |
|---|---|---|
| 0–1s | Avatar fades in (opacity 0→1) | C4 note |
| 1–2.5s | Channel name slides up (+20px → 0, opacity 0→1) | E4, G4 notes |
| 2.5–4s | Tagline fades in below name | C5 note (resolves) |
| 4–6s | Subscribe button pulses in ("訂閱頻道 ▶" on amber pill) | Gentle sustain, fade out |
| 6–20s | Static hold on final frame | Silent (YouTube end screen cards appear here) |

## TOML Config Changes

New fields in `configs/youtube_channels.toml`:

```toml
[profiles.ideal-parents-tw]
# ... existing fields unchanged ...
tagline      = "陪你走過每個育兒時刻"
outro_enabled = true
```

`tagline` is the one-line phrase shown under the channel name in the outro.  
`outro_enabled` defaults to `false` when absent — opt-in per profile.

## Outro Builder (`builder.py`)

`build_outro(profile_name, config, output_path, aspect_ratio)` — pure function, no pipeline state.

### FFmpeg pipeline

1. **Background**: amber gradient via `lavfi` color source (`#fff8f0` → `#ffe4c4`), resolution from `aspect_ratio` (matches storyboard — `16:9` → 1920×1080, `9:16` → 1080×1920)
2. **Avatar**: `profile.png` scaled to ~200px, circular mask via `alphaextract` + `alphamerge`, overlaid center-top, `fade=in:st=0:d=1`
3. **Channel name**: `drawtext` filter, Noto Sans CJK TC, bold, `#6b3f00`, starts at y+20, animates to final position over 1–2.5s via `y` expression
4. **Tagline**: `drawtext` filter, smaller size, `#a06030`, `alpha` expression fades 0→1 over 2.5–4s
5. **Subscribe pill**: `drawbox` (amber fill) + `drawtext` ("訂閱頻道 ▶", white), `alpha` fades 0→1 over 4–6s
6. **Static hold**: last frame frozen to 20s via `tpad=stop_mode=clone:stop_duration=14`
7. **Music ident**: 4 `sine` audio sources (262Hz, 330Hz, 392Hz, 523Hz), each 0.8s with 0.1s attack/decay, `amix` + `volume=-18dB`, fades out at 6s via `afade=out:st=5.5:d=0.5`

Output codec: `libx264 -crf 18 -preset fast`, `aac -b:a 128k`, `yuv420p`, `moov_atom` at front.

### CLI

```bash
# Build outro for a channel (re-run to update)
pipeline outro build --profile ideal-parents-tw

# Check outro status across all profiles
pipeline outro status
```

`pipeline outro build` auto-fetches `profile.png` from the YouTube Data API (`channels.list?part=snippet&id=<channel_id>` → `snippet.thumbnails.high.url`) if the file doesn't exist locally. If `channel_id` is blank in the TOML, falls back to asking the operator to drop the file manually.

## Publish Preflight Integration (`stage.py`)

Outro attachment happens in `PublishStage.run()` **before** `run_preflight()` is called — so preflight validates the already-merged file (correct size, correct duration). `run_preflight` itself is unchanged.

`PublishStage.run()` already receives `channel_config: ChannelConfig`, so it has everything it needs:

```python
# Pseudo-code — in PublishStage.run(), before run_preflight()
outro_path = Path("configs/channels") / channel_config.name / "outro.mp4"
if channel_config.outro_enabled:
    if outro_path.exists():
        final_with_outro = ctx.work_dir / "compose" / "final_with_outro.mp4"
        ffmpeg_concat([ctx.final_video_path, outro_path], final_with_outro)
        ctx.final_video_path = final_with_outro
        ctx.save()
        logger.info("publish.outro_attached", outro=str(outro_path))
    else:
        logger.warning("publish.outro_missing", expected=str(outro_path))
        # Non-blocking: upload proceeds without outro
```

`ffmpeg_concat` uses the concat demuxer (stream-copy, no re-encode) — fast and lossless.

`ctx.save()` persists the updated `final_video_path` to `context.json` so re-runs are idempotent.

## Error Handling

| Scenario | Behaviour |
|---|---|
| `profile.png` missing at build time | Auto-fetched from YouTube API via `channel_id`; if `channel_id` blank, exits with clear error |
| `outro.mp4` missing at publish time | Warning logged, upload continues without outro |
| FFmpeg concat fails | `PreflightError` raised, upload blocked, error shown |
| `outro_enabled` absent from TOML | Treated as `false`, outro silently skipped |

## Open Questions (resolved)

- **Music fallback**: if user dislikes the sine arpeggio, remove music by setting `outro_music=false` in TOML and rebuilding. No code change required.
- **Aspect ratio**: builder takes `aspect_ratio` param so it works for both landscape (16:9) and portrait (9:16) projects. Defaults to `16:9`.
